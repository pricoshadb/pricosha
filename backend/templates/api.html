Methods

  • Use case 1: View public content

    GET /public_content

    View public content posted in the last 24 hours

    Parameters:

    (Optional) page: Returns paginated results. Default page 1
    (Optional) results_per_page: Number of results per page. Default 10

    Example response:

    [
    {
        "email":"AA@nyu.edu",
        "file_path":"https://pricosha.drew.hu/img/none",
        "item_id":9,
        "item_name":"new",
        "post_time":"Wed, 05 Dec 2018 04:46:55 GMT",
        "tagged":["temp","vals"]
    },
    ...
]

  • Use case 2: Login

    POST /login

    Logs user in. Returns error if user does not exist or incorrect password

    Parameters:

    (Required) email: Account email
    (Required) password: Password for account

    Example response:

    {'Successfully logged in Nervous Nelly'}

  • POST /logout

    Logs user out.

  • Use case 3: View shared content items and info about them

    GET /get_shared_content

    View shared content with user. Returns error if user is not logged in

    Parameters:

    (Optional) page: Returns paginated results. Default page 1
    (Optional) results_per_page: Number of results per page. Default 10

    Example response:

    [
    {
        "email": "AA@nyu.edu",
        "email_member": "AA@nyu.edu",
        "email_owner": "AA@nyu.edu",
        "fg_name": "family",
        "file_path": null,
        "is_pub": 0,
        "item_id": 1,
        "item_name": "Whiskers",
        "post_time": "Wed, 05 Dec 2018 04:35:38 GMT"
    },
    ...
]

  • Use case 4: Manage tags

    GET /get_proposed_tags

    Get proposed tags e.g. where tagee is current user and status is false

    Parameters:

    (Optional) page: Returns paginated results. Default page 1
    (Optional) results_per_page: Number of results per page. Default 10

    Example response:

    [
    {
        "email_tagged": "FF@nyu.edu",
        "email_tagger": "AA@nyu.edu",
        "item_id": 1,
        "status": 0,
        "tag_time": "Wed, 05 Dec 2018 18:51:26 GMT"
    },
    ...
]

  • POST /modify_proposed_tag

    Get proposed tags e.g. where tagee is current user and status is false

    Parameters:

    (Required) email_tagger: Email of tagger
    (Required) item_id: item_id of ContentItem (Required) decision: Decision about tag: accept, decline, or no decision

    Example response:

    [
    {
        "email_tagged": "FF@nyu.edu",
        "email_tagger": "AA@nyu.edu",
        "item_id": 1,
        "status": 0,
        "tag_time": "Wed, 05 Dec 2018 18:51:26 GMT"
    },
    ...
]

  • Use case 5: Post a content item

    POST /post_content_item

    Get proposed tags e.g. where tagee is current user and status is false

    Parameters:

    (Required) item_name: Name of the new ContentItem
    (Required) is_pub: Makes ContentItem public or not (Optional) image_content: Image content

    Example response:

    [
    {
        "email_tagged": "FF@nyu.edu",
        "email_tagger": "AA@nyu.edu",
        "item_id": 1,
        "status": 0,
        "tag_time": "Wed, 05 Dec 2018 18:51:26 GMT"
    },
    ...
]

  • Use case 6: Tag a content item

    POST /tag_content_item

    Tags user in ContentItem

    Parameters:

    (Required) tagee_email: Email of tagged user (Required) item_id: item_id of ContentItem

    Example response:

    {'Successfully tagged Fred Fox'}

  • Use case 7: Add friend

    POST /add_friend

    Adds friend to FriendGroup

    Parameters:

    (Required) fg_name: Name of FriendGroup to add person to. Current user must own FriendGroup
    (Required) friend_fname: First name of person to add. (Required) friend_lname: Last name of person to add

    An error will be returned if the person is already in the friend group or if more than one person with the name exists

    Example response:

    {'Successfully added Fred Fox to friend group Family'}
